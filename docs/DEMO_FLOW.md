# 演示主链路

## 一、平台端演示

1. 打开 `http://127.0.0.1:5174/login`
2. 使用 `platform / 123456` 登录
3. 展示 Dashboard
4. 说明统计卡片含义：
   `在管批次数`、`已生成追溯码`、`累计扫码次数 PV`、`累计访客人数 UV`
5. 在首页公开追溯快捷入口输入 `test-token-2026`
6. 展示产品列表
7. 展示批次列表
8. 进入批次 `BATCH20260311001`
9. 展示批次摘要、追溯码、公开链接、二维码预览
10. 点击“打开页面”进入公开追溯页

## 二、企业端演示

1. 退出平台账号
2. 使用 `testuser1 / 123456` 登录
3. 展示企业侧 Dashboard
4. 展示企业可见产品列表
5. 展示企业批次列表
6. 进入批次 `BATCH20260311001`
7. 展示追溯入口区块和业务节点信息

## 三、公开追溯页演示

1. 直接打开 `http://127.0.0.1:5173/t/test-token-2026`
2. 说明页面是消费者扫码后看到的追溯档案
3. 重点展示：
   产品信息、批次状态、参与主体、关键节点、质检信息、监管记录
4. 如需展示错误态，可打开：
   `http://127.0.0.1:5173/t/invalid-token`

## 四、统计演示

1. 访问公开追溯页 1 到 2 次
2. 返回后台首页刷新
3. 观察 `PV / UV` 统计变化

说明：

- `PV` 会累计每次访问
- `UV` 按访客去重
- 当前统计来自 Redis，不来自 SQL 静态数据

## 五、建议讲解口径

- 平台端负责全局查看和管理
- 企业端负责维护自己企业的产品与批次信息
- 公开追溯页负责给消费者或老师查看最终档案
- 追溯码 `test-token-2026` 是本项目固定主演示入口
