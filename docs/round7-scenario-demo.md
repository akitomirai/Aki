# Round 7 Scenario Demo

## 推荐答辩展示顺序

建议按下面顺序演示，逻辑最顺，也最容易体现系统价值：

1. 企业管理员创建批次
2. 在工作台快速补录追溯
3. 上传质检摘要
4. 生成二维码并打开公开页
5. 用扫码页展示消费者看到的首屏结论
6. 回到后台展示风险处理闭环与恢复检查项

---

## 一、企业管理员演示路径

### 推荐路径

1. 进入后台批次列表页
2. 切到“待处理”视角
3. 点击“新建批次”
4. 创建成功后进入批次工作台
5. 按顶部引导依次执行：
   - 补追溯
   - 上传质检
   - 生成二维码
   - 发布批次

### 演示时建议强调

- 批次列表页已经能直接告诉管理员“下一步做什么”
- 工作台顶部不是普通详情，而是任务型摘要
- 快捷操作按业务场景分组，减少找入口成本

---

## 二、消费者扫码演示路径

### 推荐路径

1. 在后台工作台生成二维码
2. 点击公开页预览
3. 先展示公开页首屏：
   - 产品名
   - 当前状态
   - 质检结论
   - 企业名
   - 批次号
   - 产地
4. 再往下展示关键过程时间线

### 演示时建议强调

- 公开页首屏先给结论，不要求用户先理解后台字段
- 状态与质检结果更醒目，适合消费者快速判断
- 时间线默认先看关键节点，避免信息过长

---

## 三、风险处理演示路径

### 推荐路径

1. 在工作台对已发布批次执行冻结或召回
2. 展示风险区中的：
   - 异常原因
   - 当前处理阶段
   - 最近处理动作
   - 恢复发布前检查项
3. 继续补：
   - 处理说明
   - 整改记录
   - 处理中标记
   - 已整改标记
4. 再展示是否满足恢复发布条件

### 演示时建议强调

- 风险不是只改一个状态，而是有处理记录和整改进度
- 恢复发布前有明确检查项，不是拍脑袋恢复
- 后台工作台和公开页都能体现风险状态

---

## 四、适合老师 / 监管人员的讲解顺序

### 最短讲法

1. 先说系统以“批次”为核心对象
2. 展示后台批次列表如何直接提示下一步
3. 展示工作台如何串起补录、质检、二维码和风险处理
4. 展示公开页如何让消费者 3 秒看懂结果
5. 最后展示风险处理闭环

### 讲解重点

- 这不是单点功能堆叠，而是围绕批次的完整闭环
- 管理员、消费者、监管三类角色都能在同一条链路上获得清晰信息
- 本轮没有扩新模块，而是把已有主链路打磨得更顺手、更适合演示

---

## 五、后台最小演示提示

本轮后台适合重点展示两个位置：

- 批次列表页：看“下一步推荐”和缺失项提示
- 批次工作台：看“下一步推荐”“动作分组”“风险检查项”

如果答辩时间紧，优先展示这两个页面，再补公开页即可。

---

## 六、推荐直接使用的演示样例

- `DEMO-ORANGE-202603-A1`：正常批次，适合扫码页展示
- `DEMO-ORANGE-202603-D1`：草稿批次，适合管理员流程展示
- `DEMO-TEA-202603-F1`：冻结批次，适合风险处理中演示
- `DEMO-RICE-202603-R1`：召回批次，适合监管 / 答辩说明
